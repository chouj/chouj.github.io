OPML 2.0 (Draft) 发布

@Chouj · Jul 19, 2007 · 1 min read

7月16号，Dave Winer 发布了OPML格式的2.0版草案，按原文的意思，该文档“未正式公布，未启用”，2.0版的OPML格式还未“最终确定”，此文档仅供讨论、参考。 目前我们所见到的OPML文件多为1.0或1.1的格式，具体可参见如下代码： <?xml version=“1.0” encoding=“UTF-8”?> <?xml-stylesheet type=“text/xsl” href=“/say/css/opml.xsl”?> OPML 2.0草案同样详细解释了如上各属性标签的含义和作用，原文中绿色标注的“ownerID”属性，是1.0版中所没有的一个新属性。 OPML 2.0何时正式发布，尚在未知阶段，相信时日不远。其发布将带来什么样的影响，各线上线下服务是否会依此调整以支持新格式，让我们拭目以待。先让我们补习一下： OPML是Outline Processor Markup Language的缩写，致力于成为一种简洁公开易读易写的文件格式，常被作为Feed阅读器和聚合器中订阅条目的文件载体。 [ 点击进入OPML 2.0 （Draft）] Powered by ScribeFire.

RSS聚合器 之 Gregarius

@Chouj · Jul 14, 2007 · 1 min read

什么是RSS聚合器？ RSS聚合器是一种以用户被动更新模式更新，可读取RSS Feed / XML并显示新闻项的程序，可以方便地在线获取、阅读和管理RSS/XML格式的信息。 Gregarius 个人认为Gregarius已经是一款相当成熟的RSS feed聚合器程序了，具备不错的用户体验，易于操作和管理。抽儿就是拿这款程序架设的RSS阅读器。特性如下： 支持 RDF, RSS, ATOM feeds 支持智能识别； 基于浏览器Web端的前台浏览和后台管理； 支持目录分类，支持标签系统，并运用了ajax技术； 支持搜索； 支持OPML； 可显示favicon； 支持已读未读分类和条目数量显示； 具有多种主题样式，支持插件； 前台样式： 后台含八个管理标签，分别为“控制面板”、“Feed管理”、“条目管理”、“设置”、“插件”、“主题”和“分类目录”，操作简单。 Demo站点： OpenRSS 话语权 Gregarius运行环境要求： Apache服务器（推荐） PHP (PHP 4.x 或 PHP 5.x, 4.3+ 推荐) 数据库 MySQL 和 SQLite Gregarius主页 | Gregarius 0.5.4 原版下载页面 | Gregarius 0.5.3 的安装说明 最后提一下抽儿在使用中的一些感受。首先，空间的访问速度和php的版本会直接影响feed添加以及OPML导入的成败，某些时候可能feed添加成功，但没有成功读取到内容，或抓取后不能实时更新，这在话语权中表现比较明显。其次，Gregarius有经过汉化的版本，搜索可得（比如这里），但汉化的不彻底，更新时间显示为乱码，可能涉及编码UTF-8/GB2312的支持和时间日期代码格式的问题。还有就是Gregarius对Feedsky的Feed不友好。 Powered by ScribeFire.
